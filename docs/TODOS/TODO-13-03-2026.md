# Workshop Demo Roadmap: Reversible Definition Replacement

**As of:** 2026-03-13

**Last updated:** 2026-03-13  
Reprioritized for workshop demo corridor and reversible definition replacement.

---

## Operational Summary

The workshop corridor must allow a user to **declare**, **replace**, and **retract** network definitions while maintaining:

- semantically honest card state  
- successor-network rebuild  
- clean disappearance of old support  
- an **inspector that visibly proves which network is active**

Hot reload is reframed as **reversible definition replacement**, not generic compiler elegance.

---

## 1. Workshop Goal and Fast Path

**Global goal:** The system must support **declaration, replacement, and retraction of network definitions without semantic residue.**

That means:

- **Users can:** declare a network definition; replace the definition; retract the definition.
- **The system must:** rebuild successor networks; remove old semantic support; preserve contribution lineage where appropriate; reflect the change accurately in the inspector.

*Until the workshop is over, the four milestones below are the only plan that really matters.*

### Four concrete stages

**Milestone 1 — Semantic Card Runtime**  
Implement for one demo corridor:

- contribution identity (per source)  
- contradiction exposure  
- visible convergence policy  
- minimal lineage preservation  

**Milestone 2 — Definition Replacement**  
Implement:

- user declares network  
- user replaces network  
- successor network built  
- old network retracts  
- no semantic residue  

**Milestone 3 — Inspector Truthfulness**  
Ensure inspector correctly shows:

- active graph  
- removed nodes  
- contribution state  
- conflicts / ambiguity  

Optimize only if needed.

**Milestone 4 — Demo Stability**  
Add only where needed for the workshop:

- minimal persistence  
- protocol simplification (demo path only)  
- performance fixes  
- small UI polish  

---

## 2. Conceptual Dependency Hierarchy

The **semantic center** of the system is the **card contribution model (A2)**. Everything else exists to support it.

```
  A2  Card semantic state (contributions, contradiction, convergence policy)
   ↓
  B2  Reversible definition replacement / retraction
   ↓
  B1  Successor network rebuild lifecycle
   ↓
  A1  GC / disposal support
   ↓
  Inspector truth surface
   ↓
  Protocol + persistence (only if needed)
```

Without A2 the system cannot implement meaningful replacement semantics.

7. **Minimal persistence** — save/reload cards; enough state for workshop; no full storage design yet.

10. **Language surface improvements** — post-workshop.

## 7. Dependency Summary

- **A2 (card semantic state)** is the semantic center; without it the system cannot implement meaningful replacement semantics.  
- **B2 (reversible definition replacement)** depends on A2; implements declare / replace / retract with no semantic residue.  
- **B1 (successor rebuild lifecycle)** depends on B2; implements build-new / retract-old / dispose / GC on connect, detach, replace, rebuild.  
- **A1 (GC)** supports B1; allows replaced networks to actually disappear.  
- **Inspector truth surface** depends on the above; must visibly prove which network is active.  
- **Protocol and persistence** are scoped to the demo corridor and minimal demo persistence only.
