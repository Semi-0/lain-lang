## 3. Architecture refactor (checklist)

Ordered list of architecture refactor items (aligned with phases A–E). **Prioritized in Phase C:** items 4–7 are the Phase C focus (protocol refactor, storage, semantic card model, card-as-construct).

1. **GC and temporal/version semantics** — propagator-only dispose; scheduler-driven cell GC; vector clock / epoch or substitutive merge for card timestamps.
2. **Card rebuild protocol** — build new network in neighbor cell, retract old premises, GC gradually (no immediate destruction).
3. **Hot reload / replacement semantics** — premise/dependency-driven successor networks; TemporaryValueSet + PremisesSource.
4. **Frontend/backend protocol refactor** — simplify decoder layer using pattern matcher + generic operators.
5. **Storage layer for persisted card content** — where and how card state is stored across reload.
6. **Semantic card model** — card as compositional unit, possibly propagator-like.
7. **Card-as-construct composition** — spawning and local composition; pass card/cell construct into neighbor.
8. **Topology efficiency** — progressive graph tracking (no full BFS per update).
9. **Topology metadata/annotation** — interested entities; annotation layer for visualization modifiers.
10. **Minimal syntax and reactive operators for demo** — branching, sugar, reactive_map/filter/reduce, zip, etc., as needed.
11. **Reflective network-card / execution-card split and meta UI** — strict separation; unified input API; multi-projectional card system; spawned card as projected card.

## 5. Dependency Summary

- **A1 (GC)** and **A2 (vector clock)** are foundational: they make safe gradual disposal and correct card timestamps possible.
- **B1 (card rebuild without immediate destruction)** and **B2 (hot reload through premise/dependency-driven successor networks)** depend on A1/A2 and directly improve demo behavior.
- **C1–C3** (card model and persistence) build storage, card-as-semantic-unit, and card-as-construct for spawning and local composition.
- **D1–D2** (topology and introspection) add efficient progressive graph tracking and metadata/annotation for interested entities.
- **E1–E3** (language surface and reactive primitives) cover syntax and reactive operators needed by the demo, with CORE-03 experiments kept capped (E3).
